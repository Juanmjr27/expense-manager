# Expense Manager Constitution

## Core Principles

### Principle 1: Simplicity First
The application must be simple, intuitive, and easy to use. Any added complexity must be justified.

### Principle 2: Data Privacy
User financial data is sensitive. Local-first approach and strong security practices are mandatory.

### Principle 3: Responsive Design
The application must provide an excellent experience on desktop, tablet, and mobile devices.

### Principle 4: Accuracy and Auditability
All expense records must be accurate, traceable, and preferably immutable once saved.

### Principle 5: Maintainability
Code must be clean, well-structured, well-documented, and easy to maintain and extend.

## Technical Stack (Non-Negotiable)

- **Frontend**: HTML5, CSS3, JavaScript (Vanilla + Bootstrap 5)
- **Backend**: Python + FastAPI
- **Database**: SQLite (development) / PostgreSQL (production)
- **Architecture**: RESTful API + Separated Frontend
- **Version Control**: Git + GitHub

## Governance

- All major changes must first update the specification (`spec.md`).
- Code must follow Python best practices (PEP 8).
- Features must be developed incrementally (MVP first approach).
- User Stories must be independent and independently testable.
- Documentation must be kept up to date.
